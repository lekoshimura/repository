Describe the differences between forks, clones, and branches. When would you use one instead of another?
- A fork is a copy of a repository on your github account. Clone is copy on your local drive and branches are labels applied to a group of commits. 
- Use fork when you want to work on a existing project hosted on a github repository without changing the original content. 
- Use clone when you want work on a local copy of a git repository and send commits to the repository.
- Use branch to create a logical group of commits.
